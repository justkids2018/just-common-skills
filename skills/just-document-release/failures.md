# 失败模式记录 (Failure Patterns)

本文档记录 `just-document-release` 技能在实际使用中遇到的失败案例和解决方案。

---

## 失败案例 1: 只更新了 README，忽略了其他文档

### 场景
AI 只更新了 README.md，但没有更新 API 文档、CHANGELOG、示例代码等。

### 表现
- 文档不一致
- 用户困惑
- 需要补充文档

### 根因
- 只关注主要文档
- 没有识别所有需要更新的文档
- 没有使用文档清单

### 解决方案
- **文档更新清单**:
  - [ ] README.md
  - [ ] API 文档
  - [ ] CHANGELOG.md
  - [ ] 示例代码
  - [ ] 配置文档
  - [ ] 部署文档
  - [ ] 故障排查文档

### 预防措施
- 在 SKILL.md 中添加文档更新清单
- 根据改动类型提供不同的清单

---

## 失败案例 2: 文档更新滞后于代码

### 场景
代码已经合并，但文档几天后才更新。

### 表现
- 文档和代码不一致
- 用户使用旧文档
- 降低信任度

### 根因
- 认为"文档可以晚点更新"
- 没有将文档作为交付物的一部分
- 时间紧迫时跳过文档

### 解决方案
- **文档同步原则** - 代码和文档必须同时更新
- **在 PR 中包含文档更新** - 不是单独的 PR
- **文档作为 DoD** - Definition of Done 包含文档更新

### 预防措施
- 在 SKILL.md 中强调：文档必须同步更新
- 在 PR 模板中添加文档更新检查项

---

## 反模式总结

### ❌ 不要做
- 只更新部分文档
- 文档更新滞后于代码
- 认为"文档不重要"

### ✅ 应该做
- 使用文档更新清单
- 代码和文档同步更新
- 文档作为交付物的一部分
