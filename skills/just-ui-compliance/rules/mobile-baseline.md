# Mobile Visual Baseline

> 只管"看不看得见"和"舒不舒服"，不管颜色。

---

## 一、字体层级规范（Typography Scale）

每一个层级都有明确的字号范围和使用场景。检测时以下表为准。

| 层级 | 字号 | 行高比 | 典型用途 |
|------|------|--------|---------|
| Display / Hero | 28–36 pt | 1.15–1.25 | 引导页大标题、首屏大字 |
| Page Title（H1）| 22–28 pt | 1.2–1.3 | 页面/屏幕主标题、导航栏标题 |
| Section Title（H2）| 18–22 pt | 1.25–1.35 | 列表组标题、卡片区块标题 |
| Card / List Primary（H3）| 16–17 pt | 1.3–1.4 | 列表项主文字、卡片标题 |
| Body（正文）| 15–17 pt | 1.4–1.6 | 正文内容、说明文字 |
| Secondary（次要）| 13–15 pt | 1.3–1.5 | 副标题、时间戳、元数据 |
| Caption / Hint（提示）| 12–13 pt | 1.2–1.4 | 输入框提示、图片说明、小注 |
| 绝对最小 | 11 pt | — | 仅允许装饰性极细字 |

### 违规判定

- 任何非装饰性文字 `< 11 pt` → **BLOCKER**（字体过小，完全不可读）
- Caption/Hint 类文字 `< 12 pt` → **HIGH**
- 层级倒置：次要文字字号 ≥ 主要文字字号 → **MEDIUM**
- 同一屏幕内同一层级字号不统一 → **MEDIUM**

## 二、按钮内文字为什么显示不出来（Button Layout Formula）

这是最常见的"字看不见"问题。根本原因：**字号太大 + padding 太大 + 按钮高度没有跟着增大**，导致文字被裁切或溢出。

### 核心公式

```
button_height >= font_size_pt × 1.4 + vertical_padding_top + vertical_padding_bottom
```

`font_size × 1.4` 是该字号的最小安全行高（含上下行间距）。

### 标准参数参考

| 按钮规格 | 字号（推荐） | 上下 padding（各） | 最小按钮高度 |
|---------|------------|-----------------|------------|
| 标准主按钮 | 17 pt | 10–14 pt | 44–52 pt |
| 大号/英雄按钮 | 17–19 pt | 14–18 pt | 52–62 pt |
| 小号按钮/Chip | 13–15 pt | 6–10 pt | 32–40 pt |
| 文字链接按钮 | 15–17 pt | 8 pt | 38–44 pt |

### 常见错误案例

**案例 1：字号太大，按钮高度没跟上**
```
字号 22pt → 安全行高 = 22×1.4 = 30.8 pt
上下 padding 各 16 pt
→ 需要按钮高度 ≥ 30.8 + 16 + 16 = 62.8 pt
实际高度 48 pt → 文字顶部被裁切 ← BLOCKER
```

**案例 2：padding 太大，把字挤出去**
```
字号 17pt → 安全行高 = 23.8 pt
上下 padding 各 22 pt（偏大）
→ 需要按钮高度 ≥ 23.8 + 44 = 67.8 pt
实际高度 56 pt → 上下文字被剪切 ← BLOCKER
```

**案例 3：按钮宽度不够，文字水平方向被截断**
```
按钮固定宽度 120 pt
字号 15pt，左右 padding 各 16 pt
可用文字宽度 = 120 - 32 = 88 pt
"立即注册" 四字 ≈ 4×15×1.2 = 72 pt → 刚好能放
"马上开始使用" 六字 ≈ 6×15×1.2 = 108 pt → 超出 20 pt ← HIGH
```

### 修复方向

1. **调小字号**：按钮标签字号降到 15–17 pt
2. **调小 padding**：上下 padding 控制在 10–14 pt
3. **调大按钮高度**：按公式重新计算最小高度
4. **按钮不固定宽度**：用 `maxWidth` 或 `wrap_content + min_width` 代替固定宽

## 三、输入框规范

| 属性 | 规范值 |
|------|--------|
| 输入框高度 | 44–56 pt（不能 < 40 pt） |
| 输入文字字号 | 15–17 pt |
| Placeholder 字号 | 与输入文字同号（不能更小） |
| 上下内边距 | 10–16 pt |
| 左右内边距 | 12–16 pt |

**违规判定**

- 输入框高度 `< 36 pt` → **BLOCKER**（字放不下）
- 输入框高度 36–43 pt 且字号 ≥ 16 pt → **HIGH**（字可能被截断）
- Placeholder 字号小于输入文字字号 → **MEDIUM**

---

## 四、卡片与列表项规范

| 属性 | 规范值 |
|------|--------|
| 单行列表项高度 | 44–56 pt |
| 双行列表项高度 | 60–72 pt |
| 卡片内边距 | 12–20 pt（四边） |
| 卡片圆角 | 12–16 pt |

### 文字不完整显示的判定

- 容器固定高度 + `overflow: hidden` → 必须验证末行文字是否完整
- 末行只露出 1–4 pt（即不到半行高度）→ **HIGH**（看起来像断字）
- 末行完全被裁掉（内容无故消失）→ **BLOCKER**

---

## 五、圆角规范

| 组件类型 | 圆角范围 |
|---------|---------|
| 标准按钮 | 10–14 pt |
| 输入框 | 8–12 pt |
| 卡片 | 12–16 pt |
| Pill/胶囊按钮 | ≥ 20 pt（推荐用 9999）|
| 图标按钮（正方形） | 8–12 pt |

**特别注意**：圆角过大（如卡片圆角 32 pt）会把角落内容裁掉，导致靠边文字被圆角盖住 → **HIGH**

---

## 六、间距网格规范

- 基本单位：**8 pt**
- 所有 padding / margin / spacing 应为 4 的倍数（4, 8, 12, 16, 20, 24, 32 pt）
- 允许个别特殊值（如 6、10），但不能大量出现非网格值
- 非网格值密集出现 → **MEDIUM**

---

## 七、触控热区规范

| 元素类型 | 最小热区 |
|---------|---------|
| 所有可点击元素 | 44 × 44 pt |
| 主流程 CTA 按钮 | 44 × 44 pt（推荐 ≥ 48 pt 高） |
| 图标按钮（视觉小） | 视觉可小，但 padding 扩展热区至 44×44 |
| 底部导航 Tab | 44 pt 高（含 padding） |

- 主流程按钮热区 `< 44×44` → **BLOCKER**
- 次要按钮/图标热区 `< 44×44` → **HIGH**

## 八、文字溢出与截断规范

### 必须有明确的截断策略

任何可能溢出的文字都必须声明处理方式，不能让框架"自动决定"：

| 情况 | 正确处理 |
|------|---------|
| 单行文字容器固定宽度 | `maxLines: 1` + `overflow: ellipsis` |
| 多行文字容器固定高度 | `maxLines: N` + `overflow: ellipsis` |
| 内容行数不确定 | 不设固定高度，让容器随内容伸展 |
| 按钮标签过长 | 减小字号 or 改为 2 行 or 缩短文案 |

### BLOCKER 级文字截断问题

1. 文字有一半高度在容器外（视觉上"腰斩"）
2. 按钮内文字完全不可见（高度不足被完全裁掉）
3. 导航栏标题消失（fixed height + 大字号 = 整行不见）
4. 输入框 placeholder 不显示（行高或高度不足）

---

## 九、安全区与布局保护

- 顶部：内容不能延伸到状态栏下（需 safeArea top）
- 底部：内容不能延伸到 Home 指示条下（需 safeArea bottom）
- 滚动列表底部：最后一项需有足够 padding（≥ 安全区高度 + 16 pt）
- 底部悬浮按钮：不能永久遮住列表最后一项内容

---

## 十、文字缩放兼容性

验证默认（100%）与放大（115%）两个挡位：

- 115% 缩放下，按钮文字是否仍在按钮内显示完整 → 若不完整 → **BLOCKER**
- 115% 缩放下，卡片末行是否被截断 → **HIGH**
- 115% 缩放下，导航栏标题是否溢出 → **HIGH**
