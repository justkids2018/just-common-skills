# 06 — Skill 与 Workflow 规范

## Skill 设计门禁（强制）

1. 每个 `SKILL.md` 必须包含三问：做什么、何时触发、完美输出是什么。
2. “做什么”必须是单一职责，禁止泛化表述。
3. “何时触发”至少给出 5 条自然语言触发短语。
4. “完美输出”必须提供至少 1 个具体输出示例。

## Skill 编写规则（强制）

1. 指令必须可测试，禁止“适当处理”等模糊语句。
2. 必须声明边界与非目标，防止越权执行。
3. 必须定义输入与输出契约。

## Workflow 门禁（强制）

1. 多阶段任务必须使用 workflow skill 编排，禁止单提示词串行硬写。
2. 必须定义阶段顺序、成功条件、失败条件。
3. 必须定义 checkpoint 与恢复策略。
4. 必须标注需要人工确认的节点（登录、授权、破坏性操作）。

## 验证规则（强制）

1. 每个 skill/workflow 变更必须通过三场景测试：正常、边缘、压力。
2. 任一场景失败，必须补充指令或示例后重测。
3. 未通过三场景测试不得标记为生产可用。

## 禁止

- ❌ 仅描述能力，不写触发条件。
- ❌ 仅写流程，不给完美输出示例。
- ❌ workflow 无 checkpoint 与恢复策略。
- ❌ 失败后跳过验证直接发布。
