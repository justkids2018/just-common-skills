# 03 — 编码规范

## 命名与结构

1. 命名语义化且一致，禁止模糊缩写。
2. 文件名与职责一致，一文件一职责。
3. 函数短小、单一职责，避免过深嵌套。

## 复杂度控制

1. 超长文件/函数/过多参数必须拆分。
2. 复杂分支重构为可读结构化流程。
3. 稳定重复再抽象，不过早抽象。

## 注释

1. 公共接口必须有用途和边界说明。
2. 注释解释意图和取舍，不复述语法。
3. TODO 必须有责任人和日期。

## 错误与安全

1. 错误显式处理，禁止静默吞错。
2. 边界层校验输入，核心层保证业务约束。
3. 敏感信息不进日志。
4. 外部输入默认不可信。

## 禁止

- ❌ 注释掉废弃代码而不删除。
- ❌ 调试残留进主分支。
- ❌ 无意义命名和无主 TODO。
